# Program of Thoughts (PoT) — minimal honest implementation

A small, faithful end-to-end implementation of **Program of Thoughts prompting**
([Chen et al., 2022](https://arxiv.org/abs/2211.12588)) using an open-source
Qwen2.5 Instruct model (1.5B by default, 3B optional).

## The idea

Chain of Thought asks the LLM to reason *and* compute in natural language — and
LLMs are bad at arithmetic. Program of Thoughts splits the two:

1. The LLM writes a **Python program** that computes the answer.
2. A real **Python interpreter** runs the program.

The model handles reasoning; the interpreter handles computation. That's the
whole paper, and that's exactly what this code does — nothing more.

## What this actually does (the "honest" part)

- Few-shot prompts Qwen to emit a `python` code block ending in `answer = ...`.
- Extracts the code, runs it in a subprocess with a timeout.
- Reads the result: `answer` if the model defined it, otherwise the last
  top-level assigned variable (small models often store the result under a
  descriptive name like `total_cost` and skip the final `answer =` line —
  reading that computed value is part of the PoT method, recovered via AST).
- Compares it numerically to the GSM8K gold answer.
- Reports the **true** accuracy plus a breakdown of failure modes
  (`no_code`, `error`, `timeout`, `no_answer_var`). Nothing is hidden or
  hand-tuned to look better.

## Measured result (honest)

Qwen2.5-1.5B-Instruct, greedy decoding, 30 GSM8K test questions:

```
Accuracy:   15/30 = 50.0%
Exec breakdown: {'ok': 30}
```

All programs executed; the 15 misses are genuine reasoning errors by the small
model (wrong formulas), not harness failures. This is expected — the PoT paper
used far larger code models; a 1.5B model lands in this range. Use `--model
Qwen/Qwen2.5-3B-Instruct` and a larger `--num-samples` for a stronger/steadier
number.

## Setup

Requires [`uv`](https://github.com/astral-sh/uv) (PyTorch has no Python 3.14
wheels yet, so we pin 3.12):

```bash
uv venv --python 3.12
uv pip install torch transformers datasets accelerate "huggingface_hub[hf_transfer]"
```

## Run

```bash
# Single demo question (downloads ~3GB model on first run):
.venv/bin/python program_of_thought.py --demo

# Evaluate on 50 GSM8K test questions:
.venv/bin/python program_of_thought.py --num-samples 50

# Use the larger 3B model:
.venv/bin/python program_of_thought.py --model Qwen/Qwen2.5-3B-Instruct --num-samples 50
```

Per-example results (raw model output, extracted code, predicted/gold answers,
execution status) are written to `results.jsonl` so you can audit every case.

## ⚠️ Security note

This script **executes code generated by a language model** in a subprocess.
That subprocess is isolated only by an OS timeout — it is *not* sandboxed. The
generated code can in principle do anything your user account can. For untrusted
or large-scale use, run it inside a container, VM, or a locked-down sandbox.

## Files

- `program_of_thought.py` — the entire implementation (~250 lines).
- `results.jsonl` — written on each eval run.
